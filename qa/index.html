[ecko_toggle style=”solid” state=”open” title=”Windows下Sublime Text 默认打开方式问题解决办法”]删除 HKEY_CURRENT_USER\Software\Classes\Applications下的Sublime_Text.exe项。 （打开注册表：regedit）[/ecko_toggle]

[ecko_toggle style=”solid” state=”closed” title=”Nginx跨域使用字体文件的配置方法”]在Nginx的conf目录下修改nginx.conf或者vhost下对应的domain conf，在server里面添加以下代码：[ecko_code_highlight language=”javascript”]location ~ .(eot|svg|ttf|woff|woff2|otf)$ {
add_header Access-Control-Allow-Origin ;
}[/ecko_code_highlight][/ecko_toggle]

[ecko_toggle style=”solid” state=”closed” title=”英文或链接过长溢出框外怎么办？”]在css中定义：word-wrap:break-word;[/ecko_toggle]

[ecko_toggle style=”solid” state=”closed” title=”使用sessionStorage、localStorage存储数组与对象”]我们可以通过JSON对象提供的parse和stringify将其他数据类型转化成字符串，再存储到storage中就可以了。请看下面的代码：[ecko_code_highlight language=”javascript”]var obj = { name:’Jim’ };
var str = JSON.stringify(obj);

//存入
sessionStorage.obj = str;
//读取
str = sessionStorage.obj;
//重新转换为对象
obj = JSON.parse(str);
[/ecko_code_highlight]
localStorage也一样，只是和sessionStorage的存储时间不一样。
JS中的数组本质上也是对象类型，所以上面的代码对数组也是适用的。
[/ecko_toggle]

[ecko_toggle style=”solid” state=”closed” title=”安装sass时遇到gem的镜像错误”]这是由于gem的镜像在国内不能用的原因导致的，修改镜像地址为淘宝的就可以了。

[ecko_code_highlight language=”shell”]$ gem sources –remove https://rubygems.org/
$ gem sources -a https://ruby.taobao.org/
$ gem sources -l[/ecko_code_highlight]

这时出现如下提示时说明成功了：[ecko_code_highlight language=”shell”] CURRENT SOURCES

https://ruby.taobao.org[/ecko_code_highlight]

再执行下面的命令试试是不是可以了：

[ecko_code_highlight language=”shell”]sudo gem install sass[/ecko_code_highlight][/ecko_toggle]

[ecko_toggle style=”solid” state=”closed” title=”如何去掉textarea右下角的灰色角标？”]在css中定义：resize: none; ，这个样式同时禁用textarea调整大小[/ecko_toggle]
